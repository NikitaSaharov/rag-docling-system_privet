## Легенда типов вопросов

1. **Определение + формула** - вопрос требует дать определение термина и формулу расчёта
2. **Формула расширенная** - вопрос о сложной формуле с пояснением всех переменных
3. **Расчёт с данными** - вопрос содержит конкретные числа для вычисления
4. **Определение + примеры** - определение с практическими примерами из документа
5. **Определение системы** - описание концептуальной системы или методологии
6. **Неполная информация** - в документах нет полного ответа на вопрос
7. **Неполные данные** - для расчёта не хватает данных, нужно запросить
8. **Определение концептуальное** - философское или методологическое определение
9. **Формула базовая** - простая формула с пояснением
10. **Список + контекст** - перечень с дополнительным контекстом

## Критерии оценки ответов системы

### ✅ Правильный ответ должен содержать:

1. **Точность** - информация только из документов (без выдумки)
2. **Полноту** - все аспекты вопроса раскрыты
3. **Формат** - без markdown (**, ##, *), только обычный текст
4. **Формулы** - точно как в документе, с расшифровкой переменных
5. **Нормы** - если есть в документе, обязательно указать
6. **Вычисления** - при наличии данных выполнить расчёт
7. **Уточнения** - 2-3 релевантных вопроса в конце

### ❌ Недопустимо:

1. Выдуманная информация (галлюцинации)
2. Фразы "в документе говорится", "согласно контексту"
3. Подстановка произвольных чисел в формулы
4. Отказ от вычислений при наличии данных
5. Markdown форматирование в ответе пользователю

## Как использовать эту таблицу

### Для методолога:

1. **Базовое тестирование:**
   - Задайте системе вопросы из столбца "Вопрос"
   - Сравните ответ с "Ожидаемым ответом"
   - Оцените по критериям выше

2. **Расширенное тестирование:**
   - Перефразируйте вопросы своими словами
   - Добавьте вариации (например, "объясни что такое..." вместо "что такое...")
   - Проверьте работу с опечатками

3. **Документирование:**
   - Записывайте расхождения между ожидаемым и фактическим ответом
   - Отмечайте галлюцинации (выдуманные факты)
   - Фиксируйте отличное поведение системы

### Для разработчика:

1. **Регрессионное тестирование:**
   - После изменения промпта прогоните все 10 вопросов
   - Зафиксируйте, улучшились или ухудшились ответы
   - Документируйте изменения

2. **Отладка поиска:**
   - Проверьте, находятся ли правильные чанки для каждого вопроса
   - Посмотрите score релевантности
   - Проверьте работу keyword boosting и definition boosting

3. **A/B тестирование:**
   - Сравните ответы с разными температурами (0.0 vs 0.3)
   - Проверьте разные модели (DeepSeek vs другие)
   - Оцените влияние изменения system_prompt
